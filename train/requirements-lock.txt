# ============================================================================
# Pinned environment for SDXL LoRA training of the Soybrary dataset.
#
# These versions are deliberately pinned. Mismatched torch/CUDA/python and an
# unstable sd-scripts checkout are the most common causes of broken setups.
#
# TARGET ENVIRONMENT
#   Python:      3.10.x        (sd-scripts is tested on 3.10; 3.11/3.12 untested)
#   GPU:         NVIDIA A100 80GB (Lambda Labs gpu_1x_a100 / gpu_1x_a100_sxm4)
#   CUDA:        12.4
#   sd-scripts:  git tag v0.10.6  (stable release BEFORE the v0.11.0 refactor)
#
# PyTorch is NOT installed via this file. Install it first with the correct
# CUDA wheel index, then `pip install -r requirements-lock.txt`:
#
#   pip install torch==2.6.0 torchvision==0.21.0 \
#       --index-url https://download.pytorch.org/whl/cu124
#
# Then clone + install sd-scripts at the pinned tag:
#
#   git clone https://github.com/kohya-ss/sd-scripts.git
#   cd sd-scripts && git checkout v0.10.6
#   pip install --use-pep517 -r requirements.txt
#
# `accelerate config` answers for a single A100:
#   - This machine
#   - No distributed training
#   - NO  (run on CPU only?)
#   - NO  (torch dynamo?)
#   - NO  (DeepSpeed?)
#   - all (GPU ids)
#   - bf16  (A100 supports bf16; preferred over fp16)
# ============================================================================
